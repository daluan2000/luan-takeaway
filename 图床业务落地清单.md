# 图床业务（pig-media）落地清单

> 目标：支持图片上传、相册创建/管理、相册分享；后端完成图片存储与元数据管理。
> 原则：先 MVP 上线，再做异步处理与性能增强。

---

## 0. 先定 3 个关键决策（开工前）

- [ ] **存储方案**：优先复用现有 `pig-common-oss`，默认 MinIO/S3 兼容。
- [ ] **URL 策略**：是否启用 CDN 域名（建议预留 `cdnBaseUrl` 配置）。
- [ ] **分享权限策略**：公开链接是否可设置提取码、失效时间、访问次数上限。

## 3. 数据库表设计（MVP 最小集）

> 建议新建迁移脚本：`db/migration/pig_media_init.sql`（或按你现有 SQL 管理方式合并）。

- [ ] `media_file`：图片元数据
  - `id, owner_id, bucket, object_key, origin_name, content_type, size, md5, width, height, status, created_time`
- [ ] `media_album`：相册
  - `id, owner_id, name, description, cover_file_id, visible_scope, created_time`
- [ ] `media_album_item`：相册与图片关系
  - `id, album_id, file_id, sort_no, created_time`
- [ ] `media_album_member`：相册协作者
  - `id, album_id, user_id, role(owner/editor/viewer), created_time`
- [ ] `media_share_link`：分享链接
  - `id, share_token, owner_id, album_id, code, expire_at, max_view_count, current_view_count, status`
- [ ] `media_share_access_log`：分享访问日志
  - `id, share_id, visitor_ip, ua, access_time`

索引建议：

- [ ] `media_file(owner_id, created_time)`
- [ ] `media_album(owner_id, created_time)`
- [ ] `media_album_item(album_id, sort_no)`
- [ ] `media_share_link(share_token)` 唯一索引

## 4. API 设计（先做这些就能上线）

统一前缀建议：`/media`

### 4.1 图片

- [ ] `POST /media/files/upload`：上传图片（表单上传）
- [ ] `POST /media/files/presign`：获取预签名上传地址
- [ ] `POST /media/files/confirm`：前端直传后确认入库
- [ ] `GET /media/files/page`：我的图片分页查询
- [ ] `DELETE /media/files/{id}`：删除图片

### 4.2 相册

- [ ] `POST /media/albums`：创建相册
- [ ] `PUT /media/albums/{id}`：修改相册
- [ ] `GET /media/albums/page`：相册分页
- [ ] `DELETE /media/albums/{id}`：删除相册
- [ ] `POST /media/albums/{id}/items`：添加图片到相册
- [ ] `DELETE /media/albums/{id}/items/{fileId}`：移除图片
- [ ] `PUT /media/albums/{id}/items/sort`：图片排序

### 4.3 分享

- [ ] `POST /media/shares`：创建分享链接
- [ ] `DELETE /media/shares/{id}`：关闭分享
- [ ] `GET /media/shares/{token}`：分享详情（匿名可访问）
- [ ] `POST /media/shares/{token}/verify`：校验提取码（若启用）

### 6.1 复用 `pig-common-oss`

- [ ] 业务层封装 `MediaStorageService`（内部调用 `FileTemplate`）

- [ ] `images/{userId}/{yyyy}/{MM}/{dd}/{uuid}.{ext}`
- [ ] DB 记录 `bucket + object_key`，避免只存完整 URL

- [ ] 白名单类型：jpg/jpeg/png/webp/gif
- [ ] 单文件大小上限（如 10MB）
- [ ] 可选：内容哈希去重（`md5`）

## 11. 风险与规避

- [ ] **风险：对象存储权限泄露** -> 统一通过后端签发预签名 URL，短时有效
- [ ] **风险：图片外链失控** -> 分享链接可撤销、可过期、可限次
- [ ] **风险：越权访问相册** -> 所有写操作强制 owner/member 校验
- [ ] **风险：DB 膨胀** -> 访问日志按月分表或定期归档
