# 🎯 一、完整业务流程（必须闭环）

一个最小外卖流程应该是：

```
商家入驻
  ↓
商家上架菜品
  ↓
客户下单
  ↓
客户支付
  ↓
商家接单
  ↓
骑手接单配送
  ↓
订单完成
```

只要这条链路打通，就是一个完整系统。

---

# 一、商家服务（精简版）

只保留 3 个核心能力：

## 1. 商家入驻

* 提交店铺信息
* 管理员审核通过

## 2. 店铺信息管理

* 店铺名称
* 营业状态（营业 / 打烊）

## 3. 商家接单

* 查看待接单订单
* 接单操作


---

# 二、菜品服务（精简版）

只保留 3 个能力：

## 1. 菜品管理

* 添加菜品
* 修改菜品
* 删除菜品
* 上架 / 下架

## 2. 简单库存

* 设置库存
* 下单扣减库存


---

# 三、订单服务（核心闭环）

这是系统核心，保留最基本状态流转：

## 1. 下单

* 选择菜品
* 生成订单
* 计算总金额

## 2. 订单状态流转

最小状态：

```
待支付
已支付
已接单
配送中
已完成
已取消
```

## 3. 查询订单

* 用户查看订单
* 商家查看订单
* 骑手查看订单


---

# 四、支付服务（极简版）

只做：

## 1. 模拟支付

* 点击支付
* 修改订单状态为已支付

（可以先不用接真实第三方）

---

# 五、配送服务（极简版）

只保留：

## 1. 骑手注册

* 审核通过即可接单

## 2. 接单

* 查看“待配送订单”
* 点击接单

## 3. 配送完成

* 修改订单状态为已完成

---

# 🧩 六、最终精简后的服务职责

| 服务   | 只做这些      |
| ---- | --------- |
| 商家服务 | 入驻 + 接单   |
| 菜品服务 | CRUD + 库存 |
| 订单服务 | 下单 + 状态流转 |
| 支付服务 | 模拟支付      |
| 配送服务 | 接单 + 完成   |
